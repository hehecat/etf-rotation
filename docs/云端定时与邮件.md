# 云端定时信号 + 邮件

## 结论怎么选

| 方案 | 适合 | 不适合 |
|------|------|--------|
| **GitHub Actions** ✅ | 定时跑 Python、取数、发邮件、存仓位 | — |
| **GitHub Pages** | 手机浏览器看 `latest` 静态页 | 不能跑策略 |
| **Vercel** | 前端展示 | 免费档定时弱、跑 npx/长任务别扭 |

**推荐组合**: Actions 每天算信号并发邮件 +（可选）Pages 展示。

---

## 一次性配置

### 1. 建 GitHub 仓库并推送

```bash
cd /home/abc/etf-rotation
git init
git add config etf_rotation scripts .github docs README.md .gitignore .env.example
git add output 2>/dev/null || mkdir -p output && echo '{}' > output/.gitkeep
git commit -m "feat: ETF rotation C01 cloud-ready"
# 在 GitHub 新建空仓库后:
git remote add origin git@github.com:你的用户名/etf-rotation.git
git branch -M main
git push -u origin main
```

### 2. 配置 Secrets (仓库 → Settings → Secrets and variables → Actions)

| Secret | 示例 | 说明 |
|--------|------|------|
| `SMTP_HOST` | `smtp.qq.com` | QQ/163/Gmail SMTP |
| `SMTP_PORT` | `465` | SSL 端口; 587 则用 STARTTLS |
| `SMTP_USER` | `xxx@qq.com` | 登录邮箱 |
| `SMTP_PASSWORD` | 授权码 | **不是**登录密码 |
| `MAIL_TO` | `xxx@qq.com` | 收件人, 多个逗号分隔 |
| `MAIL_FROM` | 可选, 默认同 USER | |
| `MAIL_SUBJECT` | `ETF轮动信号` | 可选 |

#### QQ 邮箱授权码
1. QQ 邮箱 → 设置 → 账户 → POP3/SMTP  
2. 开启 SMTP, 生成授权码  
3. `SMTP_HOST=smtp.qq.com` `SMTP_PORT=465`

#### 163 邮箱
- `SMTP_HOST=smtp.163.com` `SMTP_PORT=465`

#### Gmail
- 需应用专用密码; `SMTP_HOST=smtp.gmail.com` `SMTP_PORT=465`

### 3. (可选) 开 GitHub Pages
Settings → Pages → Build and deployment → Source: **GitHub Actions**  
跑成功后会有 `https://你的用户.github.io/仓库名/`

### 4. 手动试跑
Actions → **Daily ETF Signal** → **Run workflow**  
看日志是否生成 `latest.txt`、是否发信。

### 5. 关掉本机 cron (可选)
```bash
crontab -l   # 确认
crontab -e   # 注释掉 run_daily / run_etf_signal
```

## 仓位如何跨天保留

Actions 每次是新机器, 所以流程是:

1. checkout 仓库里的 `output/模拟仓位.json`
2. 跑信号、更新仓位
3. **commit + push** 回仓库

这样模拟账户在云端连续。

## 本地测发信

```bash
export SMTP_HOST=smtp.qq.com SMTP_PORT=465
export SMTP_USER=xxx@qq.com SMTP_PASSWORD=授权码
export MAIL_TO=xxx@qq.com
export ETF_DATA_DIR=/home/abc/etf-rotation/output
# 先有一份 latest.txt
cp ~/桌面/ETF轮动信号/latest.txt output/
python3 scripts/send_email.py
```

## 注意

1. **公开仓库**不要提交邮箱授权码; Secrets 仅存 GitHub  
2. 公开仓库 push 的 `latest.txt` 谁都能看 → 敏感可改 **private** 仓库  
3. 免费 Actions 有分钟配额, 本任务每天 ~2–5 分钟, 一般够用  
4. `stock-api` 依赖外网; 若某天取数失败, 看 Actions 日志  
5. C01 仍冻结: 云端只执行, 不自动调参
