# 工程结构说明

## 目标

把原先散落在 `/home/abc/etf_*.py` 与桌面目录的脚本，收敛为可维护工程：

- 配置与代码分离
- 信号 / 回测 / 取数 / 因子 模块化
- 生产策略 C01 冻结，升级时少踩坑

## 包职责

| 模块 | 职责 | 何时改 |
|------|------|--------|
| `config/*.json` | 策略参数、ETF池 | 调参 / 加减标的 |
| `data.py` | stock-api 取数与缓存 | 换数据源 |
| `factors.py` | 效率/动量/Z分 | 加因子 |
| `backtest.py` | 历史回测引擎 | 改交易规则仿真 |
| `portfolio.py` | 模拟账户决策与下单 | 改止损/迟滞/调仓窗逻辑 |
| `signal.py` | 生产信号编排 | 改日报流程 |
| `report.py` | 文本/JSON 输出 | 改展示 |
| `paths.py` | 路径常量 | 改输出目录 |

## 数据流

```
pool.json + c01.json
        │
        ▼
   data.fetch_many  ──►  market bars
        │
        ▼
 factors.build_etf_table  ──►  scores / ranked
        │
        ▼
 portfolio.decide + execute  ──►  模拟仓位.json
        │
        ▼
 report + latest.txt/json
```

## 兼容

- 旧 cron `/home/abc/run_etf_signal.sh` → 转发新 `run_daily.sh`
- 新 cron: `30 15 * * 1-5 /home/abc/etf-rotation/scripts/run_daily.sh`
- 人工阅读仍在 `~/桌面/ETF轮动信号/latest.txt`
- 旧脚本在 `archive/`

## 升级 checklist

1. 改逻辑只动对应模块，不复制整文件
2. C01 `frozen: true` 期间不改 weights/rb/stop
3. 新实验写 `scripts/` 或临时 notebook，结果进 `archive/research_runs`
4. 回测验证: `python3 scripts/run_backtest.py --count 2500 --adjust qfq`
